# 90 秒演示视频脚本

目标：用 60 到 90 秒证明项目是一个完整的 FPGA + Linux 边缘巡检感知节点，而不是单点实验。

## 录制前检查

- FPGA 已下载 `top_ov2640_ddr3_udp_preview` bitstream。
- 树莓派服务已启动：`sudo systemctl restart edge-node.service`。
- Web 页面可访问：`http://树莓派IP:5000/`。
- 页面显示 `ONLINE`、`800 x 600`、摄像头预览画面。
- `低阈值演示` 前先确认 `buzzer_on` 已打开。

## 时间轴

### 0-10 秒：项目定位

画面：Web 控制台全屏，能看到状态卡片和摄像头预览。

口播：

```text
这是我做的基于 FPGA 和 Linux 的边缘智能巡检感知节点。摄像头数据先进入 FPGA，FPGA 负责采集、DDR3 缓冲、ROI 统计和告警判断，树莓派 Linux C++ 服务负责网络、控制、日志和 Web 展示。
```

### 10-25 秒：系统在线和真实输入

操作：

- 指向 `Node State=ONLINE`。
- 指向 `Resolution=800 x 600`。
- 指向摄像头实时画面。

口播：

```text
这里是一路真实 OV2640 摄像头输入，当前是 800x600 RGB565。页面不是播放静态图片，而是 FPGA 通过 UDP 回传的最近帧，Linux 服务重组后给 Web 显示。
```

### 25-45 秒：ROI 框选和参数下发

操作：

- 在预览画面上拖拽框选一块区域。
- 展示 `ROI_X/Y/W/H` 自动变化。
- 点击 `下发当前 ROI`。

口播：

```text
我可以直接在预览图上框选巡检区域，页面会把显示坐标换算成 FPGA 的 ROI 寄存器参数。点击下发后，参数通过 Linux 服务经 UDP 命令通道写入 FPGA。
```

### 45-65 秒：统计变化和告警触发

操作：

- 指向 `Bright Count` 和 `ROI Sum`。
- 点击 `低阈值演示`。
- 等待 ROI 框、预览边框、Alarm 卡片变红，蜂鸣器响约 0.5 秒。

口播：

```text
ROI 内的亮点数和亮度累加是在 FPGA 侧按帧统计的。现在我把告警阈值调低，亮点数超过阈值后，FPGA 输出 alarm_active，页面同步变红，蜂鸣器会响约半秒。
```

### 65-80 秒：事件留痕

操作：

- 滚动到告警事件表。
- 展示时间、Frame、ROI、Bright、ROI Sum。
- 点击快照 `查看`。

口播：

```text
告警不是只在页面上闪一下，Linux 服务会在告警上升沿记录事件，并保存当时的快照。这里能看到触发时间、帧号、ROI、亮点数阈值和 ROI 亮度累加。
```

### 80-90 秒：总结

画面：回到 Web 控制台整体。

口播：

```text
所以这个项目覆盖了 FPGA 采集预处理、DDR3/FIFO 缓冲、UDP 通信、Linux C++ 服务、Web 控制和告警闭环，适合面向 FPGA、嵌入式 Linux C++ 和边缘智能硬件方向展开。
```

## 录制注意

- 不要把时间花在 Vivado 或编译过程上，视频只展示最终系统。
- 低阈值演示前，先把 ROI 框到有亮度变化的区域。
- 如果现场环境太亮导致一直告警，就把 `alarm_count_threshold` 调高后再开始录制。
- 如果蜂鸣器声音太小，视频里重点展示页面告警和事件表，蜂鸣器作为辅助效果即可。
