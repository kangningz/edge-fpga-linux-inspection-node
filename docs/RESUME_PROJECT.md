# 简历项目写法

## 项目名

基于 FPGA + Linux 的边缘智能巡检感知节点

## 一句话版本

基于 ACX750/XC7A100T FPGA、OV2640 摄像头和树莓派 5，设计并实现一个具备摄像头采集、DDR3 帧缓存、FPGA 轻量预处理、UDP 传输、Linux C++ 服务、Web 展示、参数热更新和告警控制能力的端侧巡检感知节点。

## 简历三条版

- 基于 `OV2640 + XC7A100T FPGA + Raspberry Pi 5` 搭建边缘智能巡检感知节点，完成 DVP 摄像头采集、DDR3 RGB565 帧缓存、UDP/RGMII 图像预览回传和 32 字节遥测上报，实现真实输入到 Web 展示的系统闭环。
- 设计 FPGA 侧预处理和控制链路，支持帧边界识别、ROI 亮度累加、亮点计数、阈值告警、约 0.5 秒蜂鸣器方波提示，以及 `ROI / bright_threshold / alarm_count_threshold / tx_mode` 等寄存器热更新。
- 使用 C++ 在树莓派 5 上实现多线程服务，完成 UDP 遥测接收、命令下发、RGB565 预览重组、HTTP 控制台、日志、配置热重载和 systemd 部署，可实时展示 `800x600` 预览画面和告警状态。
- 实现 Web 端 ROI 可视化框选、告警事件表和快照留痕，支持在预览图上拖拽选择巡检区域，并将页面坐标换算为 FPGA 运行参数后在线下发。

## 偏 FPGA 岗版本

- 完成 OV2640 SCCB 初始化与 DVP 时序采集，处理 `VSYNC/HREF/PCLK` 帧行同步和 RGB565 像素组包。
- 基于 DDR3/MIG 构建整帧缓存路径，并通过 UDP/RGMII 将 RGB565 预览帧分片回传到 Linux 侧。
- 设计跨时钟域数据通路，使用异步 FIFO 和同步寄存器连接 `camera_pclk / eth_clk125m / sys_clk`。
- 实现命令接收寄存器组，支持运行时修改 ROI、阈值、发送模式和告警使能。

## 偏嵌入式 Linux C++ 岗版本

- 在树莓派 5 上使用 C++ 实现多线程边缘节点服务，包含 UDP 接收线程、HTTP 服务线程和 watchdog 线程。
- 设计 JSON 状态接口和 Web 控制台，支持实时预览、在线状态、遥测字段、运行参数和告警状态展示。
- 实现命令下发 API，支持采集开关、蜂鸣器开关、默认参数应用和运行参数批量热更新。
- 支持配置文件热重载、日志记录、systemd 部署和离线超时判定。

## 面试展开点

FPGA：

- OV2640 初始化和 DVP 采集。
- DDR3/MIG 帧缓存写读调度。
- RGB565 字节序、分片回传和预览重组。
- RGMII TX/RX 和 RX 相位问题定位。
- 跨时钟域 FIFO、脉冲同步、状态同步。
- 告警保持时间为什么不影响主数据链路。
- 无源蜂鸣器为什么需要方波驱动，而不是告警期间输出恒定电平。

Linux：

- 多线程状态同步和 mutex 使用。
- UDP 协议解析与命令包封装。
- `/api/status` 如何组织遥测、预览和运行参数。
- Web 页面如何避免缓存导致预览不刷新。
- Web 页面如何把拖拽框选坐标换算为 FPGA ROI 参数。
- systemd 部署和长期运行稳定性。

系统：

- 为什么 FPGA 做统计，Linux 做服务和展示。
- 为什么当前选择规则型轻量检测，而不是复杂深度学习。
- 为什么 DDR3 和 FIFO 在系统里承担不同职责。
- 如何从问题定位体现工程能力。

## 可量化结果

- 摄像头输入：`SVGA 800x600 RGB565`。
- 有效像素统计：`active_pixel_count=480000`。
- 状态接口：`/api/status` 返回在线状态、状态位、预览信息、运行参数和告警信息。
- 预览格式：Linux 侧将 FPGA 回传的 RGB565 数据转换为 BMP 供 Web 显示。
- 告警闭环：`bright_count >= alarm_count_threshold` 后 `alarm_active=1` 并联动蜂鸣器。
- 告警留痕：事件表记录时间、帧号、ROI、`bright_count/threshold`、`roi_sum` 和快照链接。
